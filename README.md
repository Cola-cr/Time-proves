# 项目介绍
## 项目背景
当我们在处理案件的时候，物证是一个重要的证据，有的时候，一张照片就可以视作一个不在场证明。
## 功能介绍
用户输入一张照片，请你帮他设置一个时间证据，这个证据需要有法律作用。如我们可以将照片发布到几个大平台（如哔哩哔哩，小红书，微博）法律可以认定为照片是在发布前拍的。还可以截取某股的股价或其他无法预测的信息，以确认这张照片不是提前准备的。并返回股价或其他无法预测的信息和发布的网址。
# 软件详细介绍

本软件用于“时间与证据生成”场景，帮助你采集网页信息、生成证明报告（HTML）。

---

## 主要功能

- 证据采集与发布信息整理
  - 在“发布网址”文本区粘贴或输入多行 URL，系统会自动收集并用于报告生成。
- 生成证明报告（HTML）
  - 一键生成包含时间戳、来源说明、发布链接等信息的 HTML 报告。
  - 报告将输出到本地输出目录（默认：`output/`），支持快速打开。
  - 关键操作与异常会写入日志区域，便于定位问题。

## 使用指南

1. 发布信息填写
   - 在“发布网址”文本区逐行输入或粘贴目标链接。
   - 点击“生成证明报告(HTML)”将会创建包含上述链接的 HTML 报告。

2. 打开输出目录
   - 点击“打开输出目录”按钮，快速定位已生成的报告文件夹。

3. 预览 Markdown 文件
   - 在“Markdown预览（只读）”区域点击“打开 .md 文件”。
   - 选择本地 `.md` 文件后，系统将以富文本方式渲染预览。
   - 预览支持：标题、加粗、斜体、行内代码、代码块、链接等基础语法。

4. 查看日志
   - 底部日志区会显示操作状态与错误信息，方便排查问题。

---

## Markdown 预览说明

- 支持语法：
  - 标题：`#`、`##`、`###`
  - 加粗：`**text**`
  - 斜体：`*text*` 或 `_text_`
  - 行内代码：`` `code` ``
  - 代码块：行以三反引号 ``` 包围的内容
  - 链接：`[text](url)`（仅变色显示，不内置点击跳转）
- 只读模式：
  - 预览区默认不可编辑，避免误改原文档。
- 已知限制：
  - 暂不支持嵌套复杂语法与表格、列表等完整渲染；后续版本将逐步完善。

## 常见问题（FAQ）

1) 打开 `.md` 文件后没有富文本效果？
- 请确认文件为 UTF-8 编码；确认使用了标准 Markdown 语法；若仍无效，请查看日志区是否有异常信息。

2) 生成报告失败？
- 检查发布网址列表是否为空；确保输出目录可写；查看日志区错误信息。

3) 如何更改输出目录？
- 当前版本默认固定在项目下的 `output/`，如需自定义请在后续版本中提供配置项或修改源码对应常量。
